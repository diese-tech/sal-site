# SAL Design System

## North Star

SAL should feel like a Discord-native competitive esports platform with mythic neon serpent energy.

The UI should avoid looking like:

- fantasy football software
- generic admin dashboards
- trading card clones
- stat-sheet scouting tools

## Visual Identity

### Mood

- dark
- cinematic
- social
- competitive
- serpent/order themed
- stream-readable
- energetic during key moments

### Color Direction

Base:

- near-black
- deep navy
- charcoal graphite

Glow/accent:

- cyan
- teal
- electric blue
- magenta
- purple plasma

Impact/active states:

- solar orange
- ember red
- hot coral
- bright gold/white highlights

Use red/orange sparingly for active draft tension, confirmation states, timer danger, and draft lock moments.

## Typography Direction

Preferred direction:

- clean modern sans-serif for UI readability
- slightly sharper display face for headings/logo moments
- avoid over-fantasy fonts that hurt legibility

The app should feel premium and competitive before it feels decorative.

## Motion Direction

Motion should be flashy during draft moments but restrained during normal browsing.

Default UI:

- soft hover lift
- subtle glow
- smooth opacity transitions
- light glass/frosted panels

Draft lock moments:

- selected player pulses red before confirmation
- player burns/smokes out of pool after lock
- player reappears into roster slot through smoke/glow
- active org shifts with a short delay on mobile spectator view

## Component Tone

Cards should feel closer to Discord profile cards than esports trading cards.

Use:

- avatar-first identity
- banner art
- compact metadata
- role pills
- status badges
- subtle borders/glows

Avoid:

- large rarity frames
- giant stat blocks
- fake card-game borders
- excessive decoration

## Role Styling

Smite 2 Conquest roles:

- Solo
- Jungle
- Mid
- Carry / ADC
- Support

Optional tag:

- Flex / Fill

Role styling should use readable pills/icons. Color can help, but text labels must remain present for clarity.

## Accessibility / Readability

The live board must be readable:

- on Twitch
- in screenshots
- on desktop spectator view
- on mobile spectator view

Do not sacrifice roster readability for visual effects.
